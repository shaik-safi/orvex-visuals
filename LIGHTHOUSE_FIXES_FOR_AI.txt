FIX 1 — NAVBAR MOBILE MENU ACCESSIBILITY
FILE: components/Navbar.tsx
================================================================================

PROBLEM
  Lighthouse reported that the mobile menu button had no accessible name.
  The button also did not expose expanded/collapsed state to assistive tech.

TARGET AREA
  The button near the right side of the navbar that toggles `isOpen`.

CHANGE 1A — REPLACE THE MOBILE MENU BUTTON

OLD:
  <button onClick={() => setIsOpen(!isOpen)} className={mobileToggleClasses}>
    {isOpen ? <X size={22} /> : <Menu size={22} />}
  </button>

NEW:
  <button
    onClick={() => setIsOpen((prev) => !prev)}
    className={mobileToggleClasses}
    aria-label={isOpen ? "Close navigation menu" : "Open navigation menu"}
    aria-expanded={isOpen}
    aria-controls="mobile-navigation"
  >
    {isOpen ? <X size={22} /> : <Menu size={22} />}
  </button>

CHANGE 1B — ADD AN ID TO THE MOBILE NAVIGATION CONTAINER

OLD:
  <div className={`lg:hidden overflow-hidden transition-all duration-500 ease-out ${isOpen ? "max-h-96 opacity-100" : "max-h-0 opacity-0"}`}>

NEW:
  <div id="mobile-navigation" className={`lg:hidden overflow-hidden transition-all duration-500 ease-out ${isOpen ? "max-h-96 opacity-100" : "max-h-0 opacity-0"}`}>

WHY THIS CHANGE EXISTS
  - `aria-label` gives the icon-only button a readable name.
  - `aria-expanded` tells screen readers whether the menu is open.
  - `aria-controls` links the button to the controlled menu region.
  - `setIsOpen((prev) => !prev)` is slightly safer than closing over old state.

FIX 3 — HOMEPAGE HERO SHOULD NOT DOWNLOAD ALL HERO IMAGES IMMEDIATELY
FILE: components/home/ClientSections.tsx
================================================================================

PROBLEM
  The Hero component originally rendered all hero images at once using:

    {images.map((img, i) => (
      <Image ... className={i === currentImage ? "opacity-100" : "opacity-0"} />
    ))}

  Even though only one image was visible, all three images were present in the DOM.
  Lighthouse showed the homepage downloading multiple hero images on initial load.

GOAL
  Render only the currently active hero image, then optionally preload the others later.

CHANGE 3A — ADD currentHeroImage

Find the top of `Hero()`.

OLD:
  export function Hero() {
    const [currentImage, setCurrentImage] = useState(0)
    const images = IMAGES.hero
    const { next } = getUpcomingMonths()

NEW:
  export function Hero() {
    const [currentImage, setCurrentImage] = useState(0)
    const images = IMAGES.hero
    const currentHeroImage = images[currentImage] ?? images[0]
    const { next } = getUpcomingMonths()

CHANGE 3B — KEEP THE ROTATION TIMER
  Leave the existing image rotation effect in place:

    useEffect(() => {
      const timer = setInterval(() => {
        setCurrentImage((prev) => (prev + 1) % images.length)
      }, 5000)
      return () => clearInterval(timer)
    }, [images.length])

CHANGE 3C — ADD DELAYED PRELOAD FOR THE REMAINING HERO IMAGES

Add this new effect immediately after the interval effect:

  useEffect(() => {
    if (images.length <= 1) return

    const preloadTimer = window.setTimeout(() => {
      images.forEach((src, index) => {
        if (index === 0) return
        const image = new window.Image()
        image.src = src
      })
    }, 2500)

    return () => window.clearTimeout(preloadTimer)
  }, [images])

Why the preload is delayed:
  - The first image gets priority.
  - Remaining images are fetched later, after the initial paint is more stable.
  - This keeps the carousel behavior but reduces first-load waste.

CHANGE 3D — REPLACE THE IMAGE MAP WITH A SINGLE IMAGE ELEMENT

OLD:
  {images.map((img, i) => (
    <Image
      key={i}
      src={img}
      alt="Orvex Visuals Photography"
      fill
      className={`object-cover transition-opacity duration-1000 ${i === currentImage ? "opacity-100" : "opacity-0"}`}
      sizes="100vw"
      priority={i === 0}
      quality={i === 0 ? 85 : 75}
      placeholder="blur"
      blurDataURL="data:image/svg+xml;base64,PHN2Zy..."
    />
  ))}

NEW:
  <Image
    key={currentHeroImage}
    src={currentHeroImage}
    alt="Orvex Visuals Photography"
    fill
    className="object-cover"
    sizes="100vw"
    priority={currentImage === 0}
    quality={currentImage === 0 ? 85 : 75}
    placeholder="blur"
    blurDataURL="data:image/svg+xml;base64,PHN2Zy..."
  />

IMPORTANT
  Preserve the existing `blurDataURL` exactly.
  Preserve `fill`, `sizes`, `quality`, and `alt`.

FIX 5 — ADD SECURITY HEADERS IN next.config.mjs
FILE: next.config.mjs
================================================================================

PROBLEM
  Lighthouse reported missing security headers:
  - no Content-Security-Policy
  - no COOP
  - no frame protection / X-Frame-Options

GOAL
  Add a safe baseline set of response headers via Next config.

IMPORTANT
  Do NOT add Trusted Types here unless you validate the entire app afterward.
  Trusted Types can break apps if introduced blindly.

REPLACE THE ENTIRE FILE CONTENT WITH THIS:

const contentSecurityPolicy = `
  default-src 'self';
  script-src 'self' 'unsafe-inline' 'unsafe-eval';
  style-src 'self' 'unsafe-inline';
  img-src 'self' data: blob: https://images.unsplash.com;
  font-src 'self' data:;
  connect-src 'self' https://vitals.vercel-insights.com;
  frame-ancestors 'none';
  object-src 'none';
  base-uri 'self';
  form-action 'self';
  upgrade-insecure-requests;
`.replace(/\s{2,}/g, " ").trim()

const securityHeaders = [
  {
    key: "Content-Security-Policy",
    value: contentSecurityPolicy,
  },
  {
    key: "Cross-Origin-Opener-Policy",
    value: "same-origin-allow-popups",
  },
  {
    key: "X-Frame-Options",
    value: "DENY",
  },
  {
    key: "X-Content-Type-Options",
    value: "nosniff",
  },
  {
    key: "Referrer-Policy",
    value: "strict-origin-when-cross-origin",
  },
  {
    key: "Permissions-Policy",
    value: "camera=(), microphone=(), geolocation=()",
  },
]

/** @type {import('next').NextConfig} */
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: "https",
        hostname: "images.unsplash.com",
      },
    ],
  },
  async headers() {
    return [
      {
        source: "/(.*)",
        headers: securityHeaders,
      },
    ]
  },
}

export default nextConfig

WHY THESE EXACT VALUES WERE CHOSEN
  - `img-src` allows local images, data/blob URLs, and Unsplash.
  - `connect-src` includes Vercel analytics endpoint.
  - `same-origin-allow-popups` is a safer COOP default for sites that may open external tabs.
  - `frame-ancestors 'none'` and `X-Frame-Options: DENY` block clickjacking.
